# Visual QA Screenshot Archive

本目录用于保存前端视觉 UI 改动的验收截图。截图是视觉改动的交付证据，必须和对应代码或样式改动一起提交。

## 归档结构

每次视觉 UI 改动创建一个独立目录：

```text
docs/visual-qa/<YYYYMMDD>-<short-task>/
```

目录内至少包含：

- `README.md`：记录任务、固定 URL、视口、截图文件、验证命令和已知偏差。
- `1920-*-after.png`：固定 `1920×1080` 视口下的最终态截图。

如果改动影响多个状态，分别保存对应截图，例如：

- `1920-view-after.png`
- `1920-edit-after.png`
- `1920-empty-after.png`

## 禁止提交

不要把以下内容放入归档目录：

- 浏览器 profile。
- trace、HAR、录屏草稿和临时日志。
- 构建产物、下载缓存或调试脚本。
- 与最终验收无关的大体积临时截图。

临时截图应放在本任务专用临时目录，收尾时清理；只有最终验收截图进入本目录。
