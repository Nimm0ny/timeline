## 范围

读模式渲染器 + CM6 live preview 支持 Markdown 脚注，守 §9 读↔编辑零位移。两种形态：
- **行内引用** `[^label]` → 上标标记（就地，不跳转、不重排）。
- **定义行** `[^label]: 内容` → muted 脚注块（label 上标前缀 + 内容走行内 markdown）。

均**就地渲染**（不搬到底部、不重新编号），以保读↔编辑零位移。

## 关键设计：不走语法树，按行 regex

`@codemirror/lang-markdown` 的 Lezer **无脚注语法**，且会误解析：同行多个 `[^x]` 被合并成
一个 `Link` 节点（`a[^1]b[^note]c` → 单个 Link `[1,13]`），定义行被当 `LinkReference`。故编辑器
**不用语法树处理脚注**，改为**按可见行 regex**：
- ViewPlugin 的 `Link` 分支：凡 `[^` 开头且 **<4 个 LinkMark**（无 URL）的 Link 一律跳过（脚注，交 regex）；
  `[^x](url)`（4 个 LinkMark = 真链接）照常渲染，与读渲染器 link-first 一致。
- regex pass 处理 `[^label]` 引用与行首 `[^label]:` 定义；引用若紧跟 `(` 视为真链接而跳过。
- regex pass 跳过表格/代码/图片块区（各自块级 replace，脚注 replace 落进去会重叠令 CM 抛错）。

## 实测（playwright）

读 `.markdown-body` vs 编辑 `.markdown-live-editor`（含引用的段落 + 定义行）：

| 项 | 读 | 编辑 |
| --- | --- | --- |
| 含引用段落高 | 55.5 | 55.5（Δ0） |
| 定义行高 | 22.09 | 22.09（Δ0） |
| 引用上标 | sup 11px accent | mark 11px accent |
| 定义内容行内 md（`**粗**`/`` `码` ``） | 渲染 | 渲染（标记隐藏） |

边角一致性（read==edit）：

| 输入 | 读 | 编辑 |
| --- | --- | --- |
| `[^a](#fn-a)` | 链接（文字 `^a`） | 链接 mark（无字面 `(#fn-a)`） |
| `   [^ind]: …`（缩进） | 普通段落（`[^ind]` 上标 + `: …`） | 同（非定义块） |
| 同行多引用 `[^1]…[^note]` | 两个独立上标 | 两个独立上标（不合并） |

console 0 errors / 0 warnings。点击引用 → 显原文 `[^1]`，其余引用仍隐藏（逐引用判活）。
