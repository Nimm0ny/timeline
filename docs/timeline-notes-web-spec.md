# 时间线笔记 Web 端设计规范

> 历史参考：本文档记录早期 Web 端减法规范，不作为当前实施或验收依据。当前前端硬约束以 `AGENTS.md` 和 `docs/agent-frontend-hardness.md` 为准；视觉基准以 `docs/00-mandatory-readonly-design-brief.md` 与 `timeline_notes_pixel_perfect_1920x1080_one_view.html` 为准。

## 目标

- 产品定位：自用历史事件 timeline 笔记，不是研究后台，不是 CMS。
- 页面结构：左栏分类，中栏时间线，右栏详情与编辑。
- 设计原则：少层级、少悬浮、少无意义状态，优先让时间线和内容本身可读。

## 视觉基调

- 参考：Obsidian 的三栏信息密度和克制感，不做营销化包装。
- 唯一主元素：中栏年份数字。它是页面里唯一明确高于其他层级的元素。
- 主元素比例：年份数字约为普通标题的 `1.6x`。
- 默认做减法：删掉承载不了信息的图标、统计、装饰和辅助文案。

## 颜色

- 页面底色：`#f7f2e8`
- 细网格背景线：`rgba(43, 35, 29, 0.03)`
- 主内容卡片：`#fffdf8`
- 次级卡片/输入底：`#f2e7d8`
- 主文字：`#211b17`
- 次级文字：`#73665d`
- 弱提示文字：`#a29284`
- 分割线：`#dfcfbf`
- 唯一强调色：`#bb2d24`
- 不再定义第二强调色；标签、次级控件、边框全部回到中性色。

## 字体

- 统一字体：`"Segoe UI", "PingFang SC", "Microsoft YaHei", system-ui, sans-serif`
- 不额外引入装饰性标题字体。
- 只允许两档字重：`400` 和 `600`
- 只允许五档字号：
  - `12px`
  - `14px`
  - `16px`
  - `20px`
  - `32px`

## 圆角

- 页面大面板：`16px`
- 普通卡片：`16px`
- 输入框/按钮：`12px`
- 胶囊标签和 segmented：`999px`

## 阴影

- 只允许中性灰阴影，不允许彩色阴影。
- 主面板阴影：`0 10px 24px rgba(33, 27, 23, 0.06)`
- 普通卡片阴影：`0 4px 12px rgba(33, 27, 23, 0.05)`
- 禁用 glow、blur、玻璃态和渐变装饰。

## 尺寸

- 左栏宽度：`260px`
- 右栏宽度：`360px`
- 中栏内容最大宽度：`760px`
- 顶部搜索条高度：`42px`
- 主按钮高度：`40px`
- 时间线卡片最小高度：`128px`
- 时间节点日期块宽度：`92px`

## 真实视口

- 桌面主工作视口：`1440px`
- 常规笔记本：`1280px`
- 三栏降两栏断点：`1100px`
- 两栏降单栏断点：`820px`
- 所有文本必须在 `1280px` 和 `1440px` 两档都不溢出。

## 字符预算

- 左栏专题标题：最多 `22` 个汉字或 `44` 个英文字符，超出两行截断。
- 左栏专题摘要：最多 `28` 个汉字或 `56` 个英文字符，一行截断。
- 中栏卡片标题：最多 `24` 个汉字或 `48` 个英文字符，两行截断。
- 中栏卡片摘要：最多 `72` 个汉字或 `140` 个英文字符，三行截断。
- 标签文本：最多 `8` 个汉字或 `16` 个英文字符，超出省略。
- 右栏详情正文不截断；编辑态 textarea 负责容纳长文本。

## 状态

- 加载态：
  左栏显示“正在加载专题...”，中栏显示“正在加载时间线...”，右栏不显示占位骨架，只显示空说明。
- 空态：
  没有专题时，左栏提示先新建专题；筛选无结果时，中栏提示当前筛选条件下没有记录；未选中事件时，右栏提示从中栏选择一条事件。
- 错误态：
  页面主体区必须能显示错误文案，不能只靠 toast。错误态优先出现在中栏主区域。
- 保存态：
  编辑保存失败只显示一条错误提示，不做二次弹窗。

## 深浅色

- 默认浅色主题先落地。
- 同一套 token 必须预留深色值，主题切换靠 token，不靠重写组件结构。
- 深色主题沿用相同强调色 `#bb2d24`，禁止新增第二强调色。
- 深色主题只调整背景、边框、文字层级，不改变信息架构。

## 布局规则

- 左栏只放分类、主题列表和新建入口，不放设置、主题编辑、导出等次要工具。
- 中栏顶部只保留搜索、分组切换、当前主题信息和新建按钮。
- 中栏时间线使用纵向流，不再保留旧版横向轨道、hover 卡、导出布局等交互。
- 右栏默认显示详情，点击编辑后切换为同位编辑视图，不弹独立模态。
- 左栏、中栏、右栏都只允许一个主任务，不承载额外后台能力。

## 组件规则

- Sidebar Item：一行主标题 + 一行统计或日期范围。
- Timeline Card：最多 4 个元素：标题、时间、摘要、标签行。
- Detail Panel：标题、时间、主题、年代、图片、条目内容、编辑按钮。
- Editor Panel：结构化日期、标题、年代、图片、事件条目列表、保存/取消。
- Tag：中性底色，不承担分类配色职责。
- 每张卡片元素上限 `N = 4`：
  标题、时间、摘要、标签行。超过就删，不加补充统计。

## 禁止项

- 不保留独立 `/editor` 工作台。
- 不保留 theme editor、layout export、hover preview、hero 文案区。
- 不用嵌套卡片包卡片。
- 不做假指标、假统计、无意义图标栏。
- 不做渐变背景、彩色阴影、emoji 图标、glow、blur。
