# 必读只读设计文档

状态：

- `frozen`
- `read-only`
- 后续 Web 端实现必须遵守本文档

适用范围：

- 自用 `timeline` 时间线笔记产品
- 当前只约束 Web 桌面端亮色 one-view
- 当前设计基准为 `1920×1080`

唯一原型源：

- `timeline_notes_pixel_perfect_1920x1080_one_view.html`

## 1. 核心裁决

新版原型是本阶段像素级还原基准。任何旧设计文档、旧原型、旧 1536 截图或旧“减法”规则，只要与新版 one-view 原型冲突，都以新版原型为准。

当前阶段只做：

- 桌面端
- 亮色模式
- `1920×1080`
- 一屏还原

当前阶段不做：

- 深色模式
- 移动端
- 低宽度断点优化
- 后端架构重构
- 新数据迁移

已确认删除：

- 左栏日历。日历不实现，不进入像素验收。

## 2. 必须还原的 1920 热点

以下尺寸为实现和验收的硬基准，目标容差 `±4px`：

尺寸摘要：左主按钮 `253×40`，搜索 `565×40`，卡片 `531px` 宽，composer `662×77`，右编辑区 `508×797`。

| 元素 | 坐标与尺寸 |
|---|---:|
| 左栏快速记录按钮 | `x=21, y=80, w=253, h=40` |
| 搜索笔记 | `x=329, y=16, w=565, h=40` |
| 1840 鸦片战争卡片 | `x=473, y=167, w=531, h=175` |
| 1911 辛亥革命卡片 | `x=473, y=387, w=531, h=147` |
| 1949 新中国成立卡片 | `x=473, y=580, w=531, h=162` |
| 秦始皇统一六国卡片 | `x=473, y=791, w=531, h=155` |
| 底部新增记录输入框 | `x=354, y=968, w=662, h=77` |
| 右栏顶部用户栏 / 添加入口 | 当前阶段删除，不进入验收 |
| 保存 | `x=1751, y=80, w=84, h=38` |
| 右侧笔记编辑区域 | `x=1393, y=226, w=508, h=797` |

三栏宽度按原型推导：

- 左栏约 `293px`
- 中栏约 `1075px`
- 右栏约 `552px`

显示器尺寸变化规则：

- `1920×1080` 是像素级还原基准；1920 宽度下三栏固定为 `293 / 1075 / 552`。
- `.timeline-workspace` 必须占满浏览器 `100vw × 100vh`，左栏贴左、右栏贴右，不使用外层 `transform: scale()` 承载桌面布局。
- 非 1920 窗口必须用响应式变量重新计算左栏、右栏、中栏、事件卡、rail、年份列和 composer 尺寸；中栏是尺寸变化时的优先视觉区域。
- 高度变化只改变各栏内部滚动区域高度，不得通过外层裁切隐藏左栏设置、底部 composer 或右栏底部操作区。
- 当前阶段不做移动端重排；非 `1920` 窗口通过响应式 CSS 变量重算三栏、事件卡、rail、年份列和 composer，不使用外层画布缩放。
- 全项目滚动容器允许滚动但不显示滚动条，包括页面、左栏、中栏、右栏和弹层内部滚动区域。

图标规则：

- 图标必须用真实 DOM/SVG/CSS 图形，不使用截图铺底。
- 常规图标视觉尺寸锁定在 `16px~18px`，按钮热区按场景使用 `28px/32px/36px/40px`。
- 线性图标统一约 `1.65px` 线宽，圆角和端点保持 round。
- 图标尺寸以 1920 基准为准，非 1920 窗口下跟随所在组件真实布局，不随外层画布整体缩放。
- 运行时 SVG 图标必须走 Lucide 图标库，并通过 `ui/src/components/timeline-notes/TimelineLucideIcon.vue` 集中维护名称、尺寸和线宽；不允许用截图铺底，也不允许散落手写 SVG。
- 新版原型 HTML 只提供整体 PNG，未提供可直接复用的 iconfont/SVG 源；当前阶段不按 PNG 反推 SVG token，改用 Lucide 体系并按原型校准外层热区、图标视觉尺寸、线宽和颜色。
- 右栏编辑态顶部图标顺序固定为左箭头、右箭头、星标、锁、三点、保存、分隔线、关闭式阅读视图切换。

## 3. 左栏功能

左栏必须实现：

- 品牌区
- 红色“快速记录”
- 全部笔记、今天、本周、收藏、回收站
- 笔记本列表
- 标签列表
- 设置入口

视觉裁决：

- 品牌右侧菜单按钮必须还原原型的裸三行列表图标，不使用红棕描边小方按钮。
- 品牌行按原型裁图逐项校准：书本 logo 约 `x=28, y=20, w=27, h=30`；标题暗像素约 `x=76, y=25, w=112, h=20`；右侧菜单暗像素约 `x=256, y=28, w=16, h=14`；品牌区底部分割线约在 `y=68`。

功能语义：

- 所有左栏入口都做真实功能。
- 筛选计数按当前 topic 统计。
- `全部笔记`：当前 topic 内 `deletedAt == null` 的事件。
- `今天`：当前 topic 内 `deletedAt == null` 且 `createdAt` 属于本地今天的事件。
- `本周`：当前 topic 内 `deletedAt == null` 且 `createdAt` 属于本地本周的事件。
- `收藏`：当前 topic 内 `favorite == true && deletedAt == null` 的事件。
- `回收站`：当前 topic 内 `deletedAt != null` 的事件。
- `笔记本`：切换 topic。
- `标签`：在当前 topic 和当前左栏主筛选范围内按 `tags[]` 过滤。
- `设置`：打开设置面板或设置弹层。
- 筛选叠加固定为：topic 是最外层范围；`全部笔记 / 今天 / 本周 / 收藏 / 回收站` 是主筛选；标签在主筛选结果上继续叠加过滤。点击标签不清空主筛选，切换主筛选也不自动清空标签，除非当前标签在结果中不再存在。
- 搜索范围固定为当前 topic + 当前左栏筛选结果；搜索不跨 topic，也不绕过回收站/收藏/今天/本周/标签筛选。
- 标签列表来自当前 topic 事件的 `tags[]` 自动聚合，不单独建标签表。编辑事件标签后，左栏标签和计数自动更新。
- 设置入口打开应用内轻量设置弹层，包含数据导出入口、当前版本和原型基准信息；当前阶段不做单独设置页面。

删除行为：

- 删除进入回收站：设置 `deletedAt = now`。
- 回收站内支持恢复：设置 `deletedAt = null`。
- 回收站内支持永久删除：调用永久删除接口。
- 回收站内只允许查看、恢复和永久删除；不允许编辑正文、修改标签、切换收藏或上传附件。

收藏行为：

- 卡片星标和右栏星标都可以切换收藏。
- 两处操作同一个 `favorite` 字段。
- 切换后立即持久化。

## 4. 中栏功能

中栏以时间线为主体，像素优先，支持纵向滚动展示。

必须实现：

- 顶部搜索入口，居中栏左侧；默认只显示 SVG 图标，点击后展开搜索框
- 顶部时间筛选下拉控件，居中栏右侧，入口仅显示 SVG 图标
- 年份/月/纵向 rail
- 年份红色小点、月份浅色小点、rail 到卡片前的浅色横线、卡片前红色小点
- 事件卡
- 底部 composer

交互规则：

- composer 点击后直接让右栏进入新建态。
- composer 的可见结构必须按原型保留：圆形加号、竖向分隔线、占位文案、日历/图片/旗帜三个图标按钮、红色圆形发送按钮。
- 时间筛选下拉用于按事件时间定位时间线节点。
- 时间筛选控件固定为仅 SVG 的单按钮下拉 + 小面板输入。面板内提供年份、年月、日期输入/选择，确认后执行定位。不要改成常驻搜索输入，也不要只做预设列表。
- 事件时间定位支持年份、年月、具体日期。
- 定位只滚动和聚焦中栏时间线，不改变左栏筛选状态。
- 如果没有精确匹配，滚动到最近的后一个事件；没有后一个则滚动到最后一个事件。
- URL 状态支持 `?date=1840`、`?date=1840-06`、`?date=1840-06-01`。

Pretext 裁决：

- 中栏使用 `@chenglou/pretext` 做文本测量、摘要截断、卡片高度预测和时间定位 offset 预估。
- Pretext 不渲染 UI；中栏时间线、卡片、圆点、连接线、图标和 composer 必须继续是真实 DOM/CSS。
- Pretext preset 必须绑定项目内置 `TimelinePrototypeFont`，与 CSS 字体、字号、字重、行高保持一致。
- 卡片宽度基准仍是原型 `531px`；Pretext 只能稳定文本和滚动，不允许改变三栏比例或卡片视觉结构。

事件卡：

- 保留标题、摘要、标签、星标/更多操作。
- 日期信息由左侧年份/月轨道承担，不在卡片内作为单独行。
- 卡片星标可切换收藏并立即持久化。
- 卡片和右栏的 `...` 更多按钮打开同一套菜单。普通事件菜单包含：移入回收站、复制 Markdown、导出当前事件。回收站内菜单改为：恢复、永久删除。

## 5. 右栏功能

右栏默认阅读态，支持完整编辑态。

全局裁决：

- 项目当前阶段不做用户模块，右栏顶部用户头像、日历按钮和“添加时间点”入口整栏删除。
- 右栏顶部从事件 actionbar 开始，保存、编辑/阅读切换、收藏、锁定、更多等操作留在 actionbar。

阅读态：

- 默认进入阅读态。
- 标题下方必须有“日历图标 + 时间/时间范围”和“绿植图标 + 首个标签/专题 + 下拉箭头”的元信息行。
- Markdown 正文渲染为阅读排版。
- 正文无编辑框边框。
- 标签、附件、关联时间点保留为次级分组。
- 右上角提供仅图标“编辑”按钮。

编辑态：

- 点击编辑图标进入编辑态。
- 同位置按钮变成仅图标“阅读视图”按钮。
- 只做原型里的 Markdown 编辑框，不做双栏预览。
- Markdown 编辑框有明确边框。
- 工具栏、保存、标题、日期/专题、标签、附件、关联时间点按原型顺序呈现。
- 字体必须锁定为项目内置 `TimelinePrototypeFont`：`ui/src/assets/fonts/timeline-prototype-regular.ttc` 对应 400/500，`ui/src/assets/fonts/timeline-prototype-bold.ttc` 对应 600/700。该字体文件来自原型在 Windows 环境下实际落到的 `Microsoft YaHei`，不再依赖系统字体栈回退；不允许把标题、年份、卡片正文或 Markdown 区域切到宋体/衬线字体。

保存规则：

- 手动保存。
- 切换阅读态、切换事件、切换筛选或离开编辑态前，如果有未保存草稿，必须提示保存或放弃。
- 未保存提示必须使用应用内确认弹层，不使用浏览器 `confirm`。弹层操作固定为“保存 / 放弃 / 取消”：保存后继续原动作，放弃后丢弃草稿并继续原动作，取消则停留在当前编辑态。

## 6. 后端与数据契约

后端栈固定：

- FastAPI
- SQLAlchemy
- SQLite

API 路由不新增：

- `GET /api/topics`
- `GET /api/topics/:id/events`
- `POST /api/topics/:id/events`
- `PUT /api/events/:id`
- `DELETE /api/events/:id`
- `POST /api/media/upload`

事件 DTO 继续支持：

- `bodyMarkdown`
- `tags`
- `attachments`
- `relatedEventIds`
- legacy `items[]` 回退

事件 DTO 必须新增并返回：

- `createdAt`
- `updatedAt`
- `favorite`
- `deletedAt`

后端模型必须新增：

- `TimelineEvent.created_at`
- `TimelineEvent.favorite`
- `TimelineEvent.deleted_at`

## 7. 实现验收

实现必须满足：

- 不使用原型截图作为页面背景。
- 所有文字、按钮、输入、卡片、附件和关联项都是真实 DOM。
- `1920×1080` 首屏中不出现横向溢出。
- 关键热点尺寸和位置接近第 2 节表格。
- 中栏年份 rail 和卡片布局与原型一致。
- 左栏日历不出现，且不作为验收缺失。
- 右栏阅读态无编辑框边框，编辑态 Markdown 编辑框有边框。
- 左栏筛选、收藏、回收站和标签过滤都是真实功能。

验证命令：

```bash
cmd /c npm run build
cmd /c npm run test:ui
python -m pytest tests/test_timeline_api.py tests/test_date_utils.py
```

视觉验证：

- 先用 `cmd /c npm run qa:visual-fixture` 校验默认 fixture
- 使用 `cmd /c npm run qa:visual-server` 启动可追踪视觉 QA server
- 固定 URL：`http://127.0.0.1:8798/?topic=1&event=1&mode=edit`
- 固定视口：`1920×1080`
- 禁止使用旧 URL `?topic=1&event=101&mode=edit` 作为验收入口
