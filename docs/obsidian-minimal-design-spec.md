# Obsidian Minimal · 三栏时间线设计稿

状态：

- `visual-exploration`，**非生产基线**。
- 需用户视觉拍板后，才更新 `docs/00-mandatory-readonly-design-brief.md` 并进入实现。
- 配套原型：`prototypes/timeline-obsidian-minimal.html`（单文件，自适应，可直接打开）。

适用范围：自用 `timeline` 时间线笔记，桌面 Web，亮色为主、附深色。

## 1. 设计语言：为什么是 Obsidian Minimal

- 扁平、留白、发丝线分隔；几乎不用阴影，靠 `1px` 边框和层级底色区分区域。
- 强调色克制：仅用于「选中行、年代圆点、主操作、聚焦」，其余保持中性灰，越克制越高级。
- 所有命令按钮纯图标，统一 Lucide 体系（Obsidian 原生图标库即 Lucide，天然契合）。
- 正文字体沿用项目实际字体 `Microsoft YaHei`，规避通用 AI 字体味，又符合既有基准。
- 时间线是「预览/索引」，不是内容载体：中栏只给一行可扫读的预览，细节点开进右栏。

## 3. 布局：单页自适应 · 可调 · 右栏按需展开

- 整体 `position: fixed; inset: 0`，三栏 `grid-template-columns: var(--left-w) minmax(0,1fr) var(--right-w)`：左右定宽变量、中栏弹性，随窗口自适应。
- **全局禁显滚动条**：`* { scrollbar-width: none }` + `::-webkit-scrollbar { display:none }`；各栏内部独立纵向滚动。
- **三栏可拖拽**：左右栏之间有拖拽条，带 min/max（左 `220–360`，右 `360–560`）；中栏取剩余空间。
- **默认仅显示左栏 + 中栏**（`.app.right-closed` → 右栏宽度 0）。点击中栏某行 → 右栏展开为该节点详情；顶部「关闭」收起，回到两栏。展开/收起用 `grid-template-columns` 过渡。
- 中栏内容 `max-width: 1180px` 居中，常规窗口铺满，超宽屏才留白。

三栏职责：左 = 分类 / 标签管理 / 统计；中 = 列表式时间线（预览）；右 = 选中节点详情的「展开层」。

## 4. 左栏：分类 · 标签管理 · 统计（分割线分组）

自上而下用**分割线分组**（`border-top` 发丝线，对齐右栏 `.pane-sec` 风格）：视图导航 → 分类（笔记本）→ 标签管理 → 统计，组间靠分割线区隔，不做重色块。

- **视图导航**：全部笔记 / 今天 / 本周 / 收藏 / 回收站，图标 + 名称 + 计数；单选高亮。
- **分类 · 笔记本**：专题列表（近代史 / 古代史 / 党史），主标题 + 副标题 + 计数；组头悬停露出「新建」图标。
- **标签管理**：色点 + 名称 + 计数；组头常驻「排序 / 新建」图标（低透明，悬停增强）；行悬停露出「编辑」铅笔，传达“管理”而非只读。
- **统计**：三宫格（笔记总数 / 本周新增 / 收藏）+「标签分布 Top5」横向条 +「年代活动 · 每十年」迷你柱状。
- **底部固定产品设置**：左下角 `设置` 常驻（图标 + 文字）；右侧附 折叠侧栏 / 深浅色切换。

## 5. 中栏：列表 + 关联时间线（核心）

参考用户提供的设计图，将时间线做成**列表**，每行左侧挂在一条**连续脊线**上；时间线只做预览，点击行 → 右栏出详情。

结构：

- 顶部工具条：标题「历史事件」+「共 N 条」；右侧纯图标 搜索（点击展开）/ 时间定位 / 筛选 / **列设置** / **显示预览** / 新建时间点（强调色实心）。
- 吸顶列头：随激活列动态生成。
- 按 **时代（era）** 分组：分组头 = 大圆点 + 时代名 + 年份区间 · 条数（如 `近代中国 1840–1860 · 5 条`）。
- 事件行（grid 对齐）：`脊线圆点 | 时间 | 事件（标题 + 灰度预览摘要 + 附件夹子）| [地点] | [类型] | [来源] | 标签色点 | ★`。
- 脊线：每个 `.era` 一条连续竖线（`--rail`），分组头大圆点（强调色实心）、行小圆点（空心，悬停描边强调色，选中实心）。

行状态：

- 悬停：行底 `--bg-hover`，圆点描边强调色，星标淡出可见。
- 选中：左侧 `2px` 强调色竖条 + `--accent-soft` 底 + 标题转强调色 + 圆点实心。
- 收藏：星标常驻（金棕）；未收藏星标悬停可见。

**行高固定 + 显示预览（重点）**：

- 行高**固定为「不显示预览」时的紧凑间距**（`min-height: 33px`）。
- 「显示预览」（原「显示密度」更名）只控制预览摘要 `ev-sum` 的显隐（`:root[data-preview]`），**不改变行高**；预览摘要与标题同一行内联省略，开关前后行高不变。

**列可自定义（用户需求）**：

- 中栏默认以时间线为主导，列可在「列设置」浮层勾选：`时间`(必选) / `地点` / `类型` / `来源` / `标签`；切换即重算 grid 模板与列头。
- `地点 / 来源` 为**可选列，默认关闭**：当前数据模型（`TimelineEvent` 仅有 date/era/headline/body/tags/attachments）无对应字段，启用后以占位 `—` 展示，提示需补充结构化字段，绝不臆造数据。
- 分组粒度用「时代」而非「年-月」：本项目数据跨百年且稀疏，时代分组更合理；逐年/逐月分组可在数据密集的专题里另行启用。

## 6. 右栏：详情 + Obsidian 式无感编辑

- 顶部操作条（纯图标）：上一条 / 下一条 ··· 收藏 / 置顶 / 回收站 / 保存（强调色）｜ 阅读⇄编辑 / 关闭详情。
- 结构（阅读/编辑**完全一致**）：大标题 → 元信息行（日历 + 日期、绿叶 + 专题·时代 + 下拉）→ Markdown 正文 → 标签 / 附件 / 关联事件 分组。
- **无感编辑（重点，覆盖旧基准）**：取消独立编辑态 UI——**无编辑框边框、无编辑工具栏**。点击「编辑」仅把标题与正文切为可编辑（`contentEditable`），排版、尺寸、位置全程不变；`caret-color` 用强调色，正文块 hover 轻底提示可编辑。**右栏整体尺寸在阅读/编辑间不变**。原型用 `contentEditable` 演示交互；生产建议用 CodeMirror Live Preview 实现真正的 Markdown 原地编辑。
- **图片 / 附件**：正文图片**内联展示**（占位图，无真实素材时体面占位）；点击正文图片或附件行 → **Modal 放大查看**（图片灯箱 / 文本预览）。
- **关联事件**：按「同时代」聚合，点击**跳转**到该事件（中栏高亮 + 右栏切换内容）。

## 7. 图标系统（Lucide）

原型用内联 `<symbol>` sprite；实现走 `ui/src/components/timeline-notes/TimelineLucideIcon.vue` 统一维护。常规视觉尺寸 `16–18px`，线宽 `1.75`，圆角圆端。本稿新增需登记的 Lucide 名：`library`、`clock`、`filter`、`columns-3`、`align-left`（显示预览）、`check`、`maximize`、`pin`、`download`、`file`、`hash`、`calendar-search`、`panel-left`、`sliders`、`arrow-up-down`（其余已在现有网关内）。

## 8. 与冻结基准的差异（需拍板）

| 项 | 冻结基准（00-mandatory…） | 本稿 |
|---|---|---|
| 尺寸 | `1920×1080` 像素级 one-view | 单页 `100vw×100vh` 自适应、三栏可拖拽 |
| 强调色 | 红色 | Obsidian 紫（可切蓝 / 绿） |
| 中栏 | 年份大字轨 + 卡片流 | **列表 + 关联时间线脊线**，列可自定义 |
| 右栏 | 编辑态有边框编辑框 + 工具栏 | **无感编辑：无边框、无工具栏、尺寸不变** |
| 右栏默认 | 常驻三栏 | 默认两栏，点击行才展开右栏 |
| 左栏 | 视图 + 笔记本 + 标签 | 额外加入「标签管理 + 统计」，分割线分组 |
| 深色 | 当前阶段不做 | 原型已含深色（可选采用） |

以上为视觉方向变更，需用户明确同意后再改 `00-mandatory-readonly-design-brief.md`，然后落地到 `ui/src/styles/timeline-notes.css` 与对应 Vue 组件，并按既有 `qa:visual-fixture`（或新基准）视觉 QA 流程验收。
