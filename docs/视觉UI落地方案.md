# 时间线笔记 · 1920×1080 像素级视觉 UI 落地方案

> 版本：v2.1  
> 范围：Web 桌面端亮色 one-view 首屏还原  
> 唯一视觉源文件：`timeline_notes_pixel_perfect_1920x1080_one_view.html`  
> 当前阶段：只做 `1920×1080` 桌面亮色像素级还原；深色模式和移动端延后。

## 1. 目标与优先级

本阶段目标是在真实 Vue/FastAPI 产品界面中还原新版原型首屏，而不是复用旧 1536 方案或旧只读文档的减法规则。所有可见元素必须是可交互 DOM：按钮、输入框、卡片、时间线、编辑区、附件和关联时间点都要真实渲染，不允许把原型截图作为页面背景。

冲突裁决：

- 新版原型优先于所有旧文档、旧截图和旧落地方案。
- 本文档优先约束 Web 桌面亮色 `1920×1080` 视口。
- 当前不处理深色模式和移动端，不因暗色或窄屏问题牺牲桌面像素还原。
- 左栏日历不进入当前实现，即使原型中可见，也作为已确认删除项处理。
- 不新增 API 路由，不迁移旧数据结构，不引入 Pinia。

## 2. 1920 基准尺寸

原型 PNG 基准尺寸为 `1920×1080`。以下热点来自新版 HTML 中的 hotspot，实施和 QA 都按这些坐标对齐，容差目标 `±4px`。

尺寸摘要：左主按钮 `253×40`，搜索 `565×40`，卡片 `531px` 宽，composer `662×77`，右编辑区 `508×797`。

| 区域 | 坐标与尺寸 |
|---|---:|
| 左栏快速记录按钮 | `x=21, y=80, w=253, h=40` |
| 顶部搜索框 | `x=329, y=16, w=565, h=40` |
| 第一张事件卡 | `x=473, y=167, w=531, h=175` |
| 第二张事件卡 | `x=473, y=387, w=531, h=147` |
| 第三张事件卡 | `x=473, y=580, w=531, h=162` |
| 第四张事件卡 | `x=473, y=791, w=531, h=155` |
| 底部 composer | `x=354, y=968, w=662, h=77` |
| 右栏顶部用户栏 / 添加入口 | 当前阶段删除，不进入验收 |
| 保存按钮 | `x=1751, y=80, w=84, h=38` |
| 右侧笔记编辑热区 | `x=1393, y=226, w=508, h=797` |

三栏结构建议先锁定：

- 左栏：约 `293px`，右边界在 `x=293`。
- 右栏：从 `x=1368` 开始，宽约 `552px`。
- 中栏：`x=293` 到 `x=1368`，宽约 `1075px`。
- 中栏内容列不铺满，时间线卡片列以原型卡片 `531px` 宽为准。

显示器尺寸变化解决方案：

- `1920×1080` 是像素级还原基准，不是所有窗口都强制整张画布缩放。
- `.timeline-workspace` 固定占满当前浏览器 `100vw × 100vh`，左栏必须贴合浏览器左边，右栏必须贴合浏览器右边。
- 1920 宽度下三栏严格还原 `293 / 1075 / 552`；非 1920 宽度下三栏使用响应式变量计算：左栏和右栏按比例在上下限内收缩/放大，中栏取剩余宽度。
- 中栏是显示器尺寸变化时的重点区域：年份列、rail、事件卡宽度、正文测量宽度和底部 composer 必须随中栏宽度真实重排，不允许只用外层 `transform: scale()` 缩放整张界面。
- 高度变化只改变各栏内部滚动区域高度；不得通过外层裁切隐藏左栏设置、底部 composer 或右栏底部操作区。
- 当前阶段仍不进入移动端重排；非 `1920` 窗口通过响应式 CSS 变量重算三栏、事件卡、rail、年份列和 composer。
- 图标尺寸保持在当前 token 约束内，不随视口独立变大，也不依赖外层画布缩放。
- 全项目滚动容器允许滚动但不显示滚动条，包括页面、左栏、中栏、右栏和弹层内部滚动区域。

图标还原要求：

- 图标使用真实 DOM/SVG/CSS 图形，不使用截图铺底。
- 常规工具图标按原型收敛到 `16px~18px` 视觉尺寸，图标按钮热区按原型收敛到 `28px/32px/36px/40px` 对应场景。
- 线性图标统一细线风格，线宽约 `1.65px`，圆角和端点保持 round。
- 图标尺寸以 1920 基准为准，非 1920 窗口下跟随所在组件真实布局，不做独立放大或外层画布缩放。
- 运行时 SVG 图标统一使用 Lucide 图标库，通过 `ui/src/components/timeline-notes/TimelineLucideIcon.vue` 集中维护图标名称、尺寸和线宽；不得再散落手写 SVG，也不得使用 PNG 反推 SVG token 作为运行时方案。
- 新版原型 HTML 只提供整体 PNG，未提供可直接复用的 iconfont/SVG 源；因此当前阶段以 Lucide SVG 体系作为工程图标源，按原型校准外层按钮热区、图标尺寸、线宽和颜色。
- 右栏编辑态顶部按原型使用：左箭头、右箭头、星标、锁、三点、红色 `84×38` 保存按钮、分隔线、关闭式阅读视图切换图标。

## 3. 三栏还原要求

### 左栏

左栏还原原型导航密度，但明确删除日历。最终结构顺序：

1. 品牌区
2. 红色“快速记录”
3. 全部笔记 / 今天 / 本周 / 收藏 / 回收站
4. 笔记本列表
5. 标签列表
6. 设置入口

功能规则：

- 所有左栏入口都做真实功能，不做纯视觉占位。
- 筛选计数按当前 topic 统计。
- `全部笔记`：当前 topic 内 `deletedAt == null` 的事件。
- `今天`：当前 topic 内 `deletedAt == null` 且 `createdAt` 属于本地今天的事件。
- `本周`：当前 topic 内 `deletedAt == null` 且 `createdAt` 属于本地本周的事件。
- `收藏`：当前 topic 内 `favorite == true && deletedAt == null` 的事件。
- `回收站`：当前 topic 内 `deletedAt != null` 的事件。
- `笔记本`：切换 topic。
- `标签`：在当前 topic 和当前左栏主筛选范围内按 `tags[]` 过滤。
- `设置`：打开设置面板或设置弹层，不阻塞主链路。
- 筛选叠加规则固定为：topic 是最外层范围；`全部笔记 / 今天 / 本周 / 收藏 / 回收站` 是主筛选；标签在主筛选结果上继续叠加过滤。点击标签不清空主筛选，切换主筛选也不自动清空标签，除非当前标签在结果中不再存在。
- 搜索范围固定为当前 topic + 当前左栏筛选结果；搜索不跨 topic，也不绕过回收站/收藏/今天/本周/标签筛选。
- 标签列表来自当前 topic 事件的 `tags[]` 自动聚合，不单独建标签表。编辑事件标签后，左栏标签和计数自动更新。
- 设置入口打开应用内轻量设置弹层，包含数据导出入口、当前版本和原型基准信息；当前阶段不做单独设置页面。

数据规则：

- 收藏持久化到后端 `favorite` 字段。
- 回收站使用软删除字段 `deletedAt`。
- 删除进入回收站，回收站内支持恢复和永久删除。
- 回收站内只允许查看、恢复和永久删除；不允许编辑正文、修改标签、切换收藏或上传附件。

视觉要求：

- 背景是暖纸色，不做卡片化外壳。
- “快速记录”是左栏唯一强红色按钮，尺寸约 `253×40`。
- 品牌右侧菜单按钮必须还原原型的裸三行列表图标，不使用红棕描边小方按钮。
- 品牌行按原型裁图逐项校准：书本 logo 约 `x=28, y=20, w=27, h=30`；标题暗像素约 `x=76, y=25, w=112, h=20`；右侧菜单暗像素约 `x=256, y=28, w=16, h=14`；品牌区底部分割线约在 `y=68`。
- 分类行高度、图标、计数和间距按原型密度压缩。
- 日历不出现，不作为视觉验收失败。

### 中栏

中栏以时间线显示为主体，像素优先，支持纵向滚动展示。顶部同一行保留搜索和时间筛选控件；搜索入口居左，时间筛选居右，旧的年/月 segmented 控件外观不进入新版还原。

视觉要求：

- 搜索入口默认只显示 `40×40` SVG 图标按钮；点击后展开为从 `x=329` 开始、尺寸约 `565×40` 的搜索框。有搜索词时保持展开，清空后立即回到图标态。
- 时间筛选控件固定在中栏顶部右侧，入口仅显示 SVG 图标，不显示“全部时间”文字。
- 年份数字、月份、rail 和红色节点是中栏骨架；卡片不得挤压年份列。
- 时间线节点必须保留原型的小圆点层级：年份旁 rail 上红色小点、月份旁浅色小点、rail 到卡片前的浅色横向连接线，以及卡片左侧前置红色小点。
- 事件卡宽度约 `531px`，左边界约 `x=473`。
- 底部 composer 尺寸约 `662×77`，位于首屏底部。
- 底部 composer 必须还原原型内部结构：左侧圆形加号、竖向分隔线、占位文案、日历/图片/旗帜三个图标按钮、右侧红色圆形发送按钮。
- composer 点击后直接让右栏进入新建态，不做原地输入创建。

事件时间定位：

- 时间筛选下拉用于按事件时间定位时间线节点。
- 时间筛选控件形态固定为仅 SVG 的单按钮下拉 + 小面板输入：面板内提供年份、年月、日期输入/选择，确认后执行定位。不要把控件直接改成常驻搜索输入，也不要只做预设列表。
- 支持年份、年月、具体日期定位。
- 定位只滚动和聚焦中栏时间线，不改变左栏筛选状态。
- 如果没有精确匹配，滚动到最近的后一个事件；没有后一个则滚动到最后一个事件。
- URL 状态支持 `?date=1840`、`?date=1840-06`、`?date=1840-06-01`。

Pretext 中栏裁决：

- 中栏接入 `@chenglou/pretext` 作为文本测量和滚动稳定层，不作为 UI 渲染层。
- 事件卡仍使用真实 Vue DOM/CSS 渲染；时间线、圆点、连接线、图标和底部 composer 不由 Pretext 绘制。
- Pretext 只负责标题行数、摘要截断、标签行数、卡片高度预测和按事件时间定位前的滚动 offset 预估。
- 中栏 Pretext 字体 preset 必须使用项目内置 `TimelinePrototypeFont`，不得回退到 `system-ui`、宋体或其他系统字体栈。
- `1920×1080` 基准下卡片宽度锁定约 `531px`，正文测量宽度按卡片内边距推导为约 `480px`；显示器尺寸变化时卡片宽度、正文测量宽度和 Pretext 截断宽度必须同步更新。

事件卡元素：

- 标题
- 一到两行摘要
- 标签 chip
- 星标 / 更多操作

日期不放在卡片内作为单独行，日期信息由左侧年份/月轨道承担。卡片星标和右栏星标都可切换同一个 `favorite` 字段，并立即持久化。

卡片和右栏的 `...` 更多按钮打开同一套菜单。普通事件菜单包含：移入回收站、复制 Markdown、导出当前事件。回收站内菜单改为：恢复、永久删除。

### 右栏

右栏必须同时实现阅读态和完整编辑态。

全局裁决：

- 项目当前阶段不做用户模块，右栏顶部用户头像、日历按钮和“添加时间点”入口整栏删除。
- 右栏顶部从事件 actionbar 开始，保存、编辑/阅读切换、收藏、锁定、更多等操作留在 actionbar。

阅读态：

- 默认进入阅读态。
- 标题下方必须显示原型元信息行：日历图标 + 时间/时间范围，绿植图标 + 首个标签/专题 + 下拉箭头。
- Markdown 正文渲染为阅读排版。
- 阅读态正文无编辑框边框，不显示编辑态 Markdown 输入边框。
- 标签、附件、关联时间点保留为次级分组。
- 右上角提供仅图标“编辑”按钮，参考 Obsidian 单按钮视图切换。

编辑态：

- 点右上角编辑图标进入编辑态。
- 同位置按钮切换为仅图标“阅读视图”按钮。
- 编辑态按原型像素级还原：顶部工具区、保存、标题、日期/专题、Markdown 工具栏、Markdown 编辑框、标签、附件、关联时间点。
- 只做原型里的 Markdown 编辑框，不做双栏预览。
- 编辑态 Markdown 编辑框必须有明确边框。
- 保存使用手动保存。若草稿有未保存修改，切换阅读态、切换事件、切换筛选或离开编辑态前必须提示保存或放弃。
- 未保存提示必须使用应用内确认弹层，不使用浏览器 `confirm`。弹层操作固定为“保存 / 放弃 / 取消”：保存后继续原动作，放弃后丢弃草稿并继续原动作，取消则停留在当前编辑态。

视觉要求：

- 右栏顶部用户栏已删除，保存按钮位于 actionbar 内，不再按旧 `x≈1751` 顶部全局栏验收。
- 编辑正文热区从 `x≈1393, y≈226` 开始，宽约 `508px`，高约 `797px`。
- 右栏分隔线、工具栏、正文框、附件列表和关联列表按原型顺序排列。
- 右栏内容允许内部滚动，但首屏必须露出标签、附件和关联时间点。
- 字体必须锁定为项目内置 `TimelinePrototypeFont`：`ui/src/assets/fonts/timeline-prototype-regular.ttc` 对应 400/500，`ui/src/assets/fonts/timeline-prototype-bold.ttc` 对应 600/700。该字体文件来自原型在 Windows 环境下实际落到的 `Microsoft YaHei`，不再依赖系统字体栈回退；不要把标题、年份、卡片正文或 Markdown 区域改成宋体/衬线字体。

## 4. 后端与数据契约

不新增 API 路由，继续使用：

- `GET /api/topics`
- `GET /api/topics/:id/events`
- `POST /api/topics/:id/events`
- `PUT /api/events/:id`
- `DELETE /api/events/:id`
- `POST /api/media/upload`

事件 DTO 增加并返回：

- `createdAt: string`
- `updatedAt: string`
- `favorite: boolean`
- `deletedAt: string | null`

保存 payload 保持既有字段并允许写入状态字段：

- `bodyMarkdown: string`
- `tags: string[]`
- `attachments: Attachment[]`
- `relatedEventIds: number[]`
- `favorite?: boolean`
- `deletedAt?: string | null`

后端模型：

- 新增 `TimelineEvent.created_at`，旧数据可用 `updated_at` 回填。
- 新增 `TimelineEvent.favorite`，默认 `false`。
- 新增 `TimelineEvent.deleted_at`，默认 `null`。
- legacy `items[]` 继续作为兼容层保留。

## 5. 实现约束

- 使用 Vue 3 + Vite、FastAPI + SQLAlchemy + SQLite。
- 不引入 Pinia。
- 不迁移旧 `EventItem`，只作为兼容层保留。
- 不使用 glow、blur、渐变装饰或营销式 hero。
- 不把截图作为 UI，所有内容必须是 DOM。
- 深色模式和移动端不进入当前验收。

## 6. QA 验收

视觉 QA 固定使用仓库可追踪脚本：

```bash
cmd /c npm run qa:visual-fixture
cmd /c npm run qa:visual-server
```

默认状态优先打开：

```text
http://127.0.0.1:8798/?topic=1&event=1&mode=edit
```

禁止继续使用旧 URL `?topic=1&event=101&mode=edit` 作为验收入口。

验收步骤：

1. 从 `timeline_notes_pixel_perfect_1920x1080_one_view.html` 导出临时 PNG 作为人工对照图。
2. 使用 Chrome headless 或浏览器工具在 `1920×1080` 截取实现图。
3. 读取关键 DOM rect，对照第 2 节热点尺寸。
4. 核查首屏无横向滚动、无文本重叠、无空白错位。
5. 验证左栏筛选、收藏、回收站恢复/永久删除、标签过滤。
6. 验证中栏事件时间定位不改变左栏筛选状态。
7. 验证右栏阅读/编辑图标切换、手动保存和未保存提示。
8. 运行 `cmd /c npm run build`。
9. 运行前端 Node tests 和后端 pytest，确保交互数据契约未回退。

本阶段不验收：

- 深色模式
- 移动端/窄屏布局
- 旧 1536 基准截图
