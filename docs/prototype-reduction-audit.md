说明：
- 这张图是当前 web 端产品界面的原型来源。
- 本文档的任务不是“照抄”，而是判断这张图里哪些必须保留、哪些应该删掉。

## 后端技术栈决定

- 继续使用现有栈：`FastAPI + SQLAlchemy + SQLite`

理由：
- 已有代码和本地数据都围绕这套栈存在，迁移成本没有收益。
- 这是自用 timeline 笔记，不需要为并发、分布式或复杂部署提前做工程化。
- 三栏前端需要的后端能力只是结构化事件 CRUD、分类列表和图片上传，这套栈足够。

## 原型必须保留的

### 1. 三栏信息架构

- 左栏分类列表
- 中栏时间线
- 右栏详情 / 编辑

原因：
- 这是原型真正的产品骨架，不是视觉装饰。
- 左栏解决“我在看哪个专题”，中栏解决“时间顺序如何展开”，右栏解决“这一条是什么 / 怎么改”。

### 2. 中栏纵向时间线

- 年份数字
- 竖向时间线轨道
- 顺着时间排序的事件卡

原因：
- 这是唯一主元素。
- 也是区别“时间线笔记”与普通列表页的核心识别物。

### 3. 事件卡最小信息

- 标题
- 时间
- 一行摘要
- 标签行

原因：
- 这四项已经足够支持扫读、比较和点击进入详情。
- 再加任何东西都在挤占中栏主区。

### 4. 右栏详情 + 编辑切换

- 查看态：标题、时间、正文条目
- 编辑按钮
- 编辑态：日期、标题、年代、条目编辑、保存/取消

原因：
- 用户要求里明确写了“点击后出现右栏详情页，详情页可点击编辑按钮进入编辑视图”。
- 这是必须保留的任务流，不是视觉偏好。

## 原型可以砍的

### 左栏

- `Ctrl N` 热键提示
- 回收站计数
- 工作计划这类额外专题示例
- 左栏底部日历
- 设置入口

原因：
- 它们不属于当前首版三栏主流程。
- 左栏只需要承担分类导航，不该兼做“工具侧栏”。

### 顶栏

- 搜索框右侧的快捷键胶囊
- 时间过滤入口的可见文字 `全部时间`
- 头像
- 右上角多余的星标 / 锁 / 返回箭头

原因：
- 这些控件大多不承载首版信息，只会把主区注意力拉走。
- 自用产品优先键盘效率，但快捷键提示不必常驻占位。

### 中栏

- 卡片右上的星标与更多按钮
- 底部快速输入条
- 年份旁边重复的月份辅助标记，如果信息重复可以收掉一半

原因：
- 中栏是唯一主元素，次要操作不应在每张卡上重复出现。
- 事件卡如果能靠点击进入右栏完成剩余操作，就不需要在卡面堆动作。

### 右栏

- 富文本工具栏整排按钮
- 标签区里的 `+` 小按钮
- 附件区的三点菜单
- 相关时间点区的链接图标墙

原因：
- 这些都属于“编辑器外观”，不是首版必要信息。
- 首版右栏应该优先保证读写闭环，不追求工具栏完整性。

## 分组件视觉验收

### A. 左栏 Sidebar

必须保留：
- 品牌名
- 新建事件入口
- 专题列表

可删：
- 日历
- 回收站
- 设置
- 额外示例专题

验收标准：
- 左栏只看见“分类”信息，不像工具箱。
- 一屏内能辨认当前专题与专题数量。
- 没有第二视觉焦点。

### B. 中栏 Timeline Rail

必须保留：
- 大年份数字
- 时间轴轨道
- 时间排序事件卡

可删：
- 多余的卡片动作
- 底部快速输入条
- 与卡片重复的统计文本

验收标准：
- 第一眼只能落到年份数字和事件卡上。
- 主数字大约是卡片标题的 `1.5x ~ 2x`。
- 卡片之间节奏均匀，不靠阴影制造层级。

### C. 事件卡 Event Card

必须保留：
- 标题
- 时间
- 一行摘要
- 标签

可删：
- 收藏、菜单、次级操作按钮
- 多余的 metadata

验收标准：
- 单卡元素数 `<= 4`
- 卡片高度稳定
- 摘要不超过一行
- 卡片是次级，不和年份抢主元素地位

### D. 右栏详情 Detail Pane

必须保留：
- 标题
- 时间
- 正文条目
- 编辑按钮

可删：
- 大段工具图标
- 附件快捷菜单
- 装饰性状态图标

验收标准：
- 查看态以阅读为主，不像富文本软件。
- “编辑”是唯一明确动作。
- 信息顺序是：是什么 -> 什么时候 -> 写了什么。

### E. 右栏编辑态 Editor Pane

必须保留：
- 日期
- 标题
- 年代
- 条目列表
- 保存 / 取消

可删：
- 富文本工具条
- 次级插入按钮
- 非必要格式化控制

验收标准：
- 像结构化表单，不像文档编辑器。
- 控件密度高，但不显拥挤。
- 保存动作明确，取消动作退居次级。

## 当前裁剪结论

首版必须保留的是：
- 三栏布局
- 纵向时间线
- 事件卡四元素
- 右栏查看 / 编辑切换

首版建议砍掉的是：
- 日历
- 快速输入条
- 卡片级操作按钮
- 富文本工具栏
- 顶栏头像与快捷键提示
- 右栏装饰性图标群
